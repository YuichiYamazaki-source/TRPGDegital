# CoC AI-GM Discord Bot — 設計ドキュメント

## 1. プロジェクト概要

TRPGをゲーム化するプロジェクト。AI がクトゥルフ神話 TRPG (CoC) のキーパー (KP/GM) を務め、Discord チャンネル上で複数プレイヤーとテキストチャット形式でセッションを行う。

### ゴール
- 最小構成 (MVP) で動くプロトタイプを作り、実際にプレイしながら AI の品質をチューニングする
- ローカル少人数 × AI GM × 日本語という、現在市場に空白のあるポジションを狙う

### ターゲット
- 日本市場メイン
- ローカルで集まってプレイする少人数 TRPG 体験
- 将来的に Web / PC / モバイル展開を検討

## 4. MVP アーキテクチャ

```
PLたち ──chat──▶ Discord Bot ◀── scenario.md
                     │
                on_message
                     │
                     ▼
              OpenAI API (GPT-4o)
              system prompt = KPルール + シナリオ
```

### 構成ファイル（実質3つ）

1. **`bot.py`** — Discord Bot 本体
   - `on_message` で特定チャンネルの全メッセージを拾う
   - 会話履歴を Python リスト (in-memory) に append
   - OpenAI API に system prompt + 会話履歴を送信
   - レスポンスをチャンネルに返信
   - ダイスロールは Python の `random.randint(1, 100)` で生成し、テキストで結果を渡す

2. **`system_prompt.txt`** — KP としての振る舞いルール
   - チューニングの主戦場
   - KP のトーン・描写スタイル
   - CoC の基本ルール（技能判定、SAN値チェック等）
   - シナリオ情報のインジェクト

3. **`scenario.md`** — シナリオデータ
   - 導入 → 探索場所 → NPC 情報 → クライマックス条件 → エンディング分岐
   - system prompt に丸ごと、またはセクション単位でインジェクト

### MVPスコープ
- 固定シナリオ 1 本
- キャラシは手動登録 (JSON/YAML)、またはsystem promptに直書き
- 会話履歴はインメモリ（永続化は後回し）
- セッション終了 = Bot 再起動でリセット

## 5. チューニング観点

実際にプレイしながら以下を調整:

- **システムプロンプト** — KP としてのトーン、描写の濃さ、PL の自由度のバランス
- **情報開示ロジック** — 探索場所ごとにどの情報をどのタイミングで出すか
- **判定の自動化レベル** — ダイスロールを AI が判断して振るか、PL が明示的に要求するか
- **SAN値チェックの演出** — 正気度喪失の RP 描写をどこまで AI にやらせるか
- **マルチPL対応** — 誰の発言に対してどうリアクションするか、ターン制にするか自由発言か
- **記憶管理** — トークン上限対策としての要約 + 直近N件スライディングウィンドウ方式

## 6. ローカル開発セットアップ手順

1. Discord Developer Portal で Bot を作成、トークン取得
2. テスト用 Discord サーバーに Bot を招待
3. `.env` に `DISCORD_TOKEN` と `OPENAI_API_KEY` を設定
4. `python bot.py` で起動 → Bot がオンラインに
5. 指定チャンネルでチャットして動作確認

### 注意
- WSL2 の Docker Desktop 競合を避けるため、Windows 側 Python で直接実行推奨
- ローカルでは PC スリープやネットワーク切断で Bot が落ちるが、テスト段階では問題なし
- 本番運用時に Railway / Fly.io / VPS 等に移行
